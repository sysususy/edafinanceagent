# edafinanceagent 零基础复现教程

这是一份面向零基础学习者的完整教程，目标是让你在没有 NVIDIA GPU 的普通电脑上，也能把这个项目的核心部分真正复现出来。

本教程默认你的环境和我们实际验证过的环境一致：

- Windows 主机
- `WSL 2 Ubuntu`
- 没有 `NVIDIA GPU`
- 有 `OpenRouter API Key`

## 1. 先明确项目目标

这个项目不是单一脚本，而是三段式工程：

1. 多 Agent 主系统  
路径：`Financial-MCP-Agent/`

2. A 股 MCP 数据服务  
路径：`a-share-mcp-is-just-i-need/`

3. 金融新闻训练链路  
路径：根目录下的 `data_process.py`、`train_qwen_sentiment.py`、`train_qwen_risk.py`

如果你是零基础，推荐的顺序一定是：

1. 先跑通多 Agent 主系统
2. 再理解数据清洗与训练链路
3. 最后再考虑云端 LoRA / GRPO / vLLM

## 2. 为什么本教程适合无 GPU 环境

没有 NVIDIA GPU，并不意味着这个项目做不了。

你仍然可以完整完成：

- MCP 工具化
- LangGraph 多 Agent 编排
- OpenRouter 模型接入
- 新闻数据清洗
- SFT 样本构造
- Tokenizer / Label Masking 验证

你当前不适合本机做的是：

- `Qwen3-8B` 正式 LoRA 训练
- `vLLM` 本地推理部署
- `GRPO` 正式强化学习训练

因此，本教程采用两段式目标：

### 第一阶段：主系统跑通

- 能输入股票名称或代码

## 9. 修复 Baostock 并发问题

原项目在四个 Agent 并发调用时，容易出现：

- `用户未登录`

根本原因是：

- Baostock 登录态是进程级全局状态
- 多个并发登录/登出会互相打断

### 9.1 修改文件

文件：

`a-share-mcp-is-just-i-need/src/utils.py`

#### 增加导入

```python
import threading
```

#### 增加全局锁

```python
_BAOSTOCK_SESSION_LOCK = threading.RLock()
```

#### 替换 `baostock_login_context`

```python
@contextmanager
def baostock_login_context():
    """上下文管理器，处理Baostock登录和登出，抑制标准输出消息"""
    with _BAOSTOCK_SESSION_LOCK:
        original_stdout_fd = sys.stdout.fileno()
        saved_stdout_fd = os.dup(original_stdout_fd)
        devnull_fd = os.open(os.devnull, os.O_WRONLY)

        os.dup2(devnull_fd, original_stdout_fd)
        os.close(devnull_fd)

        logger.debug("Attempting Baostock login...")
        lg = bs.login()
        logger.debug(f"Login result: code={lg.error_code}, msg={lg.error_msg}")

        os.dup2(saved_stdout_fd, original_stdout_fd)
        os.close(saved_stdout_fd)

        if lg.error_code != '0':
            logger.error(f"Baostock login failed: {lg.error_msg}")
            raise LoginError(f"Baostock login failed: {lg.error_msg}")

        logger.info("Baostock login successful.")
        try:
            yield
        finally:
            original_stdout_fd = sys.stdout.fileno()
            saved_stdout_fd = os.dup(original_stdout_fd)
            devnull_fd = os.open(os.devnull, os.O_WRONLY)

            os.dup2(devnull_fd, original_stdout_fd)
            os.close(devnull_fd)

            logger.debug("Attempting Baostock logout...")
            bs.logout()
            logger.debug("Logout completed.")

            os.dup2(saved_stdout_fd, original_stdout_fd)
            os.close(saved_stdout_fd)
            logger.info("Baostock logout successful.")
```

### 9.2 为什么要这么改

因为一次完整的数据调用应该被看成：

`登录 -> 查询 -> 登出`

这个过程必须串行化，否则多个 Agent 会互相把 Baostock 会话踢掉。

## 15. 训练链路里真正重要的知识点

### 15.1 这不是普通分类头训练

情感与风险任务都被包装成了生成式对话任务：

- `System`
- `User`
- `Assistant`

模型不是输出分类 logits，而是学习生成最后的评分数字。

### 15.2 为什么要做 Label Masking

训练时不是整段文本都算 loss，而是：

- system prompt 不算
- few-shot 示例不算
- user 输入不算
- 只让最后的 `Assistant: 2` 这种答案 token 参与 loss

### 15.3 真实踩到的两个坑

#### 坑 1：token 对齐问题

例如：

- `"2"` -> `[17]`
- `" 2"` -> `[220, 17]`
- `"Assistant: 2"` -> `[71703, 25, 220, 17]`

如果只按字符位置切分，可能保留到的是空格 token，而不是真正的标签 token。

#### 坑 2：长 Prompt 截断

原模板很长：

- system prompt 很长
- few-shot 示例很多
- 新闻摘要也很长

当 `max_length=512` 时，最后的答案 `Assistant: 2` 可能直接被截掉，导致训练信号消失。

### 15.4 `left padding` 的理解

如果设置：

```python
tokenizer.padding_side = "left"
```

那么：

- 左边是 padding
- 真实文本在右边

所以判断真实 token 区间时，不能按右侧 padding 的思路处理。
